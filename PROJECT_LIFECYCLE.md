# PROJECT_LIFECYCLE.md — LETO Weekly Rhythm

This file defines the daily focus for automated lifecycle runs. Each day has a primary activity type that guides what work happens.

---

## Monday — Plan & Spec Review
**Activity:** Review `spec.md` against current implementation. Update if drifted. Check open issues/PRs. Set weekly priorities.

## Tuesday — Feature Build
**Activity:** Implement new features or significant chunks from the roadmap. Focus on `src/` code.

## Wednesday — Test & Debug
**Activity:** Run full test suite. Fix failing tests. Add missing coverage. Hunt bugs.

## Thursday — Polish & UX
**Activity:** UI/UX refinements, accessibility, responsive fixes, copy edits. Focus on user-facing quality.

## Friday — Integrate & Deploy
**Activity:** CI/CD checks, dependency updates, build verification, deployment prep.

## Saturday — Refactor & Research
**Activity:** Code cleanup, refactoring, tech debt reduction. Research new libraries or approaches.

## Sunday — Review & Document
**Activity:** Weekly wrap-up. Update documentation, README, CHANGELOG if needed. Review commits from the week. Light maintenance only — no heavy features.
